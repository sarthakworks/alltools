Optimization Techniques

  • Micro-frontends: Splitting a monolithic app into smaller, independent apps.
  • Server Side Rendering (SSR): Rendering the initial HTML on the server.
  • Reduce Latency: CDN, Caching, Compression (Gzip/Brotli).
  • Caching: Browser Cache, Service Workers.
  • Code Splitting: Load only what is needed (Lazy loading).
  • Memoization: Caching expensive function results (useMemo, React.memo).
  • Web Workers: Offload heavy processing to background threads.
  • Service Workers: Offline support, caching assets.
  • Critical Rendering: Inline critical CSS, defer non-critical JS.

Webpack

A static module bundler for modern JavaScript applications.

Key Concepts
  • Entry: The start point of the application.
  • Output: Where to emit the bundles.
  • Loaders: Transformations applied to source files (e.g., Babel for JS, css-loader for CSS).
  • Plugins: Perform wider range of tasks (bundle optimization, asset management).

Workers

Web Workers

Run in background thread. No access to DOM. Good for heavy calculations.

Service Workers

Proxy between app and network. Offline support, caching, push notifications. Requires HTTPS.

PWA (Progressive Web App)

  • Web apps that behave like native apps.
  • Requirements: HTTPS, Service Worker, Manifest File.
  • Features: Installable, Offline capabilities, Push notifications.
